## Architecture Notes

### Project Structure
```
prison-policy-ai/
├── backend/
│   ├── pipeline/          # extract, chunk, embed, query
│   ├── reports/           # classifier, generator, filler, prompts
│   ├── webapp/            # Flask app + templates
│   ├── requirements.txt
│   └── Dockerfile
├── frontend/
│   └── forms/             # Claude's 6-form React app
├── templates/             # DOC templates + parsed JSONs
├── data/                  # gitignored: PDFs, extracted, chunks, OCR
├── VISION.md              # Project vision for collaborators
├── .godplans/PLAN.mdx     # This file
└── README.md
```

### Data Flow
```
PDF → extract.py → extracted/*.txt
    → Human review → reviewed/*.txt
    → chunk.py → chunks/*.jsonl
    → gcloud storage cp → GCS
    → embed.py → Vertex AI RAG corpus
    → query.py (or webapp) → Gemini 2.5 Flash → answer + citations
```

### Report Pipeline
```
Field Notes → classifier.py → {category, forms[], charges[]}
    → generator.py → {supervisor_summary, first_person, disciplinary}
    → filler.py + 005.doc → completed DOC
    → webapp → display text + download link
```

## Running the Project

```bash
# Set up
cd backend && pip install -r requirements.txt

# RAG: extract, review, chunk, embed
cd pipeline && python extract.py && python chunk.py && python embed.py

# Query (CLI)
cd pipeline && python query.py "use of force policy"

# Web app
cd webapp && python app.py
# → http://localhost:8080

# Deploy
cd backend && gcloud run deploy prison-policy-ai --source . --region us-central1
```
