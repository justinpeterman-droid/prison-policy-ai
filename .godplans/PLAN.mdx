### Report Pipeline (v2 — Current)
```
Field Notes
    │ POST /api/reports/classify
    ▼
Classifier (Gemini) → incident_type (7 categories)
    │ officer confirms/overrides
    ▼ POST /api/reports/extract
Extraction (Gemini, temp=0, schema-constrained)
    │ → structured slots (all nullable)
    │ → find_gaps() — deterministic Python, no AI
    │ → Missing Information panel (gap questions from checklist JSON)
    ▼ POST /api/reports/generate
Generation (Gemini, structured facts ONLY)
    │ → 4 report types
    │ → auto_content inserted by code
    │ → invented_facts() scan (yellow flag on fabricated ADC#s/dates)
    ▼ POST /api/reports/download
Template Filler (python-docx) → filled 005/409 DOCX
```

### Data Flow (RAG)
```
PDF → extract.py → extracted/*.txt
    → Human review → reviewed/*.txt
    → chunk.py → chunks/*.jsonl
    → gsutil → GCS (gs://gen-lang-client-0968389176-policy-ai/chunks/)
    → import_to_agent_builder.py → Vertex AI RAG corpus
    → query.py → Gemini 3.5 Flash → answer + citations
```

## Key Design Decisions

1. **7 BMU categories** (not the old ADC categories) — `incident_checklist_v2.json` is authoritative
2. **Unit is BMU** — all generated text says "Benny Magness Unit"
3. **Extraction at temp=0, schema-constrained** — zero hallucination
4. **All slots nullable** — nulls drive the gap-question UI
5. **Reports from structured facts only** — generator never sees raw notes
6. **auto_content is code-inserted** — not AI-generated
7. **`officer_middle` → `employee_number`** — fixed in template + filler mapping
8. **Per-officer reports** — each staff gets own 005 with correct header
9. **UNKNOWN → ⚠ markers** — visible in output, never silently filled
10. **Quotes verbatim, uncensored** — evidence preservation
